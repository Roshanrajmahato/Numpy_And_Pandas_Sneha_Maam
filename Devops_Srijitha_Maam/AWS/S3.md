🔷 AWS S3 Static Website Hosting 🔷 
❓ What is Static Website Hosting ❓
Static website = Website made of:~HTML,CSS,JS,Images


👉 AWS allows hosting static websites directly from S3 bucket.

Service Used:
Amazon S3 (Simple Storage Service)

🔷 PART 1: Create & Upload Website Files
Step 1: Login to AWS Console

Go to:
👉 AWS Console → Services → S3

Step 2: Create a Bucket

Click Create bucket

Bucket name: must be globally unique
Example:

roshan-static-website-2026


Region: choose your region

Uncheck:
✔️ Block all public access

⚠️ VERY IMPORTANT:
Uncheck “Block all public access”

Click → Create bucket

🔷 PART 2: Upload Website Files
Step 3: Upload Files

Open your bucket → Click Upload

Upload:

index.html

style.css

script.js

images folder

Click → Upload

🔷 PART 3: Enable Static Website Hosting
Step 4: Enable Hosting

Go to Properties tab

Scroll down → Static website hosting

Click → Edit

Enable

Index document:

index.html


Save changes

After saving → AWS generates:

Bucket website endpoint URL


Example:

http://roshan-static-website-2026.s3-website-us-east-1.amazonaws.com

🔷 PART 4: Add Bucket Policy (MOST IMPORTANT)

By default, bucket is private.

So we must allow public read access.

Step 5: Add Bucket Policy

Go to Permissions tab

Scroll to Bucket Policy

Click Edit

Add this policy:

🔷 COMPLETE BUCKET POLICY CODE

Replace bucket-name with your bucket name.

{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "Statement1",
      "Effect": "Allow",
      "Principal": "*",
      "Action": "s3:GetObject",
      "Resource": "arn:aws:s3:::your-bucket-name/*"
    }
  ]
}


Example:

{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "Statement1",
      "Effect": "Allow",
      "Principal": "*",
      "Action": "s3:GetObject",
      "Resource": "arn:aws:s3:::roshan-static-website-2026/*"
    }
  ]
}


Click → Save Changes

🔷 Explanation of Policy (VERY IMPORTANT)
🔹 Version
"Version": "2012-10-17"


Policy language version.

🔹 Effect
"Effect": "Allow"


Allow access.

🔹 Principal
"Principal": "*"


= Everyone (public internet)

🔹 Action
"s3:GetObject"


Allow users to READ objects (HTML, CSS, JS).

🔹 Resource
"arn:aws:s3:::bucket-name/*"


Means:
All objects inside bucket.

🔷 PART 5: Test Website

Now:

Go to Properties

Copy Website Endpoint URL

Paste in browser

🎉 Website is Live!

🔷 How It Works Internally

User types URL

Browser hits S3 website endpoint

S3 checks bucket policy

If policy allows → serves index.html

Website loads

No EC2 needed.
No server needed.
Fully serverless.

🔷 Architecture Diagram (Concept)

User → Internet → S3 Website Endpoint → Bucket → index.html

🔷 Important Interview Points
🔥 Q1: Why uncheck Block Public Access?

Because S3 by default blocks public access.

🔥 Q2: What happens if bucket policy not added?

You get:

403 Forbidden

🔥 Q3: Difference between S3 Website Endpoint & S3 Object URL?

Website endpoint:

Supports index.html

Supports error.html

Object URL:

Direct object access

No routing support

🔥 Q4: Can we use HTTPS?

Direct S3 website endpoint = ❌ No HTTPS
For HTTPS:
Use:

CloudFront

ACM certificate

🔷 Optional Advanced Setup (Production)

For professional deployment:

S3 → CloudFront → Route53 → ACM (SSL)

🔷 Common Errors
❌ AccessDenied

Fix:

Check bucket policy

Check block public access

Check resource ARN
